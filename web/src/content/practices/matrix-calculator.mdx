## A calculator for в„Ө/Nв„Ө

Every value in the workspace is an integer reduced modulo **N**. Pick `N`, build matrices
with any shape you need (1Г—1 up to 6Г—6), and choose an operation. The result is either a
scalar, a matrix, or вҖ” for linear systems вҖ” a structured answer with the solution space.

The calculator is not just an evaluator: it is designed for **chained computations**. Any
result can be saved into one of ten **slots** (`S0 вҖҰ S9`). A saved slot can then be used
in a later matrix as:

- `Sk` вҖ” substitutes the full scalar (only when the slot holds a scalar),
- `Sk[i, j]` вҖ” substitutes one cell of a saved matrix (0-indexed).

This is the same mental loop you use on paper: compute an intermediate (say the inverse
of the determinant), give it a name, and plug it back in without rewriting.

## Unit pivots

Over the real numbers every nonzero element has an inverse. Over в„Ө/Nв„Ө only **units**
do вҖ” integers `a` with `gcd(a, N) = 1`. That one fact controls everything else.

| Concept | Over в„қ | Over в„Ө/Nв„Ө |
|---|---|---|
| Row scaling | divide by any nonzero pivot | pivot must be a unit |
| Matrix invertibility | `det A вү  0` | `gcd(det A, N) = 1` |
| Unique RREF | always | only when `N` is prime |
| Rank | well-defined | well-defined for `N` prime; care required otherwise |

When `N` is prime, в„Ө/Nв„Ө is a field and everything behaves like classical linear algebra.
When `N` is composite there are zero-divisors, and the calculator warns you explicitly:
results of RREF, rank, and null-space may depend on pivot choices.

## Inverse via adjugate

For a square matrix `A` with `gcd(det A, N) = 1`:

```
AвҒ»В№ вүЎ (det A)вҒ»В№ В· adj(A)   (mod N)
```

The adjugate is the transpose of the cofactor matrix. The calculator computes both and
shows the cofactor expansion step by step. The modular inverse of the determinant is
exactly the Extended Euclidean Algorithm from **Practice 01** вҖ” the same tool, applied
to one number instead of a pair.

### Worked example (Hill 2Г—2 mod 26)

Take `K = [[3, 3], [2, 5]]` over the Latin alphabet (`N = 26`):

1. `det K = 3В·5 вҲ’ 3В·2 = 9`
2. `gcd(9, 26) = 1` вҶ’ invertible.
3. `9вҒ»В№ вүЎ 3 (mod 26)` (since `9В·3 = 27 вүЎ 1`).
4. `adj K = [[5, вҲ’3], [вҲ’2, 3]] вүЎ [[5, 23], [24, 3]] (mod 26)`
5. `KвҒ»В№ вүЎ 3 В· [[5, 23], [24, 3]] вүЎ [[15, 17], [20, 9]] (mod 26)`

Verify: `K В· KвҒ»В№ вүЎ IвӮӮ (mod 26)`.

## Non-square matrices

`det`, `adj`, `AвҒ»В№`, and `Aбө–` only make sense for square matrices. Everything else
works for arbitrary `m Г— n` shapes. The calculator pivots on **RREF** for this:

- **`rref(A)`** вҖ” reduced row echelon form. Pivots are chosen among units; when no unit
  is available in a column the column is left without a pivot (a warning is emitted).
- **`rank(A)`** вҖ” number of pivot columns of `rref(A)`.
- **`solve Ax = b`** вҖ” runs RREF on `[A | b]` and reports:
  - **consistency** (no pivot in the augmented column),
  - a **particular solution** (free variables set to 0),
  - a **homogeneous basis** (one vector per free column).
- **`AвҒ»В№ right`** вҖ” a right inverse `X` with `A В· X = IвӮҳ`. Exists when `m вүӨ n` and the
  rows of `A` are linearly independent. Computed column-by-column by solving
  `A В· xвұј = eвұј`.
- **`AвҒ»В№ left`** вҖ” a left inverse `Y` with `Y В· A = IвӮҷ`. Obtained as
  `(AбөҖ right inverse)бөҖ`.

## Chaining with slots вҖ” the Hill flow

The slot bank is the core of why this practice exists. A typical session:

1. Enter the key `K` into matrix A, operation **AбөҖ** just to normalize shape вҖ” or
   directly compute whatever you need.
2. Pick `det A`, click **Compute**, and **Save вҶ’ S0**. Slot `S0` now holds the scalar
   `det K` mod `N`.
3. Build a tiny 1Г—1 matrix containing `S0`, pick **AвҒ»В№** (with N small enough), or
   just remember `S0` and compute `S0вҒ»В№` outside. (A simpler path: choose **AвҒ»В№**
   directly on `K` вҖ” the trace shows you the determinant, its inverse, and the
   adjugate in one step.)
4. Any intermediate matrix can be stored in `S1`, `S2`, вҖҰ and referenced later with
   `S1` (whole) or `S1[i, j]` (single cell).

The payoff: you never re-type a number. Every multi-step derivation becomes a short
sequence of operations whose operands are either literals or named intermediates.

## The trace

Every operation returns a structured trace: row operations (swap / scale / eliminate),
pivot placements, cofactor expansions, and free-form notes. The UI renders them in
order under the result so you can follow the derivation вҖ” the same spirit as the
affine cipher writeup, extended to matrices.

## Rust source

The logic lives in `crates/matrix/` and exposes a single `op(request_json)` function
via `wasm-bindgen`. The request is a tagged union over operations:

```rust
#[derive(Deserialize)]
pub struct OpRequest {
    pub kind: String,   // "add", "mul", "inv", "rref", "solve", ...
    pub n: i64,
    pub a: Option<Matrix>,
    pub b: Option<Matrix>,
    pub k: Option<i64>,
    pub p: Option<u64>,
    pub row_sel: Option<Vec<usize>>,
    pub col_sel: Option<Vec<usize>>,
}
```

RREF is the workhorse: it pivots on units first, falls back to any nonzero entry with a
warning, and emits every row operation into the trace.

```rust
fn op_rref(a: &Matrix, n: i64, trace: &mut Vec<Step>, warnings: &mut Vec<String>)
    -> (Matrix, Vec<usize>);
```

Inversion reuses RREF only implicitly вҖ” it goes through the adjugate path so the trace
shows cofactor expansion directly:

```rust
AвҒ»В№ = (det A)вҒ»В№ В· adj(A)   (mod N)
```

Solving `Ax = b`, right/left inverses, and rank all call RREF under the hood.

## Implementation limits

- Matrix dimensions capped at 6Г—6 in the UI to keep the trace legible.
- Powers capped at 10 000 вҖ” fast matrix exponentiation is `O(log p)` multiplications,
  but the trace becomes unreadable beyond that.
- Composite moduli emit warnings but are not refused. That is intentional: seeing
  exactly where a composite `N` breaks uniqueness is part of the lesson.
